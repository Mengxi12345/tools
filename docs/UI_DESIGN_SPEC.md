# 信息聚合产品 UI 设计规范

> 高端、现代、专业 — Bento Grid + 极简主义，适合高信息密度的聚合类产品（如今日头条 / RSS 阅读器）。

---

## 一、视觉风格定位

### 1.1 核心原则

- **Bento Grid（便当盒）**：内容以不规则网格块呈现，大块头条 + 小块列表，层次分明、扫读高效。
- **现代极简**：减少装饰线、阴影过重和纯色块堆砌；用留白和间距划分信息。
- **呼吸感**：模块间距 ≥ 24px（移动端 ≥ 16px），卡片内边距充足，避免「塞满一屏」。

### 1.2 信息层级

| 层级 | 用途 | 表现方式 |
|------|------|----------|
| 头条 / Hero | 当日或本周重点 | 大图 + 大标题 + 副标题，占首屏 40%–50% 高度 |
| 焦点区 | 次要重点 | 中等卡片，2–3 列 Bento 网格 |
| 流式列表 | 常规信息流 | 统一卡片高度或紧凑列表，无限滚动 |
| 侧边 / 底栏 | 导航、筛选、推荐 | 固定或可收起，不抢主内容 |

## 二、排版与字体

### 2.1 字体组合（中文优先）

**首选（高级无衬线）：**

- **标题 / 强调**：**思源黑体（Noto Sans SC）** 或 **阿里巴巴普惠体**  
  - 字重：700（标题）、600（小标题）、500（强调正文）
- **正文 / 长文**：**思源黑体（Noto Sans SC）** 或 **霞鹜文楷**（若希望带一点人文感）  
  - 字重：400（正文）、350（辅助说明）
- **英文/数字**：与中文同族即可；若混排多，可设 **Inter** 或 **DM Sans** 为 `font-family` 第二顺位。

**备选（系统优先、无外网字体）：**

- `"PingFang SC", "Microsoft YaHei", "Hiragino Sans GB", sans-serif`  
  - 标题用 600/700，正文 400。

### 2.2 字号层级

| 名称 | 字号 | 行高 | 字重 | 用途 |
|------|------|------|------|------|
| **H1** | 28px / 1.75rem | 1.25 | 700 | 页面主标题、Hero 主标题 |
| **H2** | 22px / 1.375rem | 1.3 | 600 | 区块标题、卡片主标题 |
| **H3** | 18px / 1.125rem | 1.35 | 600 | 列表项标题、小卡片标题 |
| **Body** | 16px / 1rem | 1.6 | 400 | 正文 |
| **Body Small** | 14px / 0.875rem | 1.5 | 400 | 摘要、次要说明 |
| **Caption** | 12px / 0.75rem | 1.4 | 400 | 时间、来源、标签、辅助信息 |

*移动端可整体缩小 1–2px，保持比例。*

### 2.3 使用规范

- 单行标题过长时使用「…」截断，或限制最多 2 行 + `line-clamp`。
- 正文段落控制在 3–5 行为佳，超出可「展开阅读」。
- 数字、时间、标签统一用 Caption 或 Body Small，颜色用辅助色。

## 三、配色方案

提供两套主题，可切换或按产品线选用。

### 3.1 浅色 — 莫兰迪色系（低饱和度）

适合「阅读感强、长时间浏览」的聚合/阅读类产品。

| 角色 | 变量名建议 | 色值 | 用途 |
|------|------------|------|------|
| 主色 | `--color-primary` | `#6B7B8C`（灰蓝） | 主按钮、链接、选中态 |
| 主色浅 | `--color-primary-light` | `#9BA8B5` | 悬停、浅底 |
| 辅助色 | `--color-accent` | `#8B9A7A`（橄榄绿） | 标签、成功、次要强调 |
| 背景 | `--color-bg-page` | `#F5F4F1` | 页面底 |
| 背景卡片 | `--color-bg-card` | `#FFFFFF` | 卡片、浮层 |
| 正文 | `--color-text-primary` | `#2C2C2C` | 标题、正文 |
| 次要文 | `--color-text-secondary` | `#6B6B6B` | 摘要、说明 |
| 辅助文 | `--color-text-tertiary` | `#9A9A9A` | 时间、来源、Caption |
| 边框 | `--color-border` | `#E5E3DF` | 分割线、边框 |
| 分割/留白 | — | 用背景色 + 间距 | 少用硬线 |

### 3.2 深色 — 霓虹点缀（Dark Mode）

适合「夜间/专业/科技感」场景。

| 角色 | 变量名建议 | 色值 | 用途 |
|------|------------|------|------|
| 背景 | `--color-bg-page` | `#0F0F12` | 页面底 |
| 背景卡片 | `--color-bg-card` | `#1A1A1F` | 卡片、浮层 |
| 背景悬浮 | `--color-bg-elevated` | `#242429` | 悬停、输入框 |
| 主色/霓虹 | `--color-primary` | `#00D4AA`（青绿） | 主按钮、链接、焦点 |
| 霓虹辅 | `--color-accent` | `#7B61FF`（紫） | 标签、高亮、次要强调 |
| 正文 | `--color-text-primary` | `#F0F0F2` | 标题、正文 |
| 次要文 | `--color-text-secondary` | `#A0A0A8` | 摘要、说明 |
| 辅助文 | `--color-text-tertiary` | `#6B6B72` | 时间、来源 |
| 边框 | `--color-border` | `#2E2E34` | 分割、边框 |

*霓虹色仅用于关键交互（按钮、链接、焦点、重要标签），面积控制在 10% 以内，避免刺眼。*

## 四、关键组件设计

### 4.1 Hero 区域（头条）

**目标**：首屏即传达「今日重点」，有视觉冲击力且不喧宾夺主。

- **布局**
  - 占首屏约 40%–50% 高度；宽屏下可左侧大图 + 右侧 1–2 条次头条。
  - 大图比例建议 16:9 或 3:2，统一圆角（如 12px），与 Bento 块风格一致。

- **内容**
  - **主标题**：H1，最多 2 行，深色模式下可加轻微文字阴影提升可读性。
  - **副标题/摘要**：Body Small，1–2 行，次要色。
  - **元信息**：Caption，来源 + 时间，置于底部或角标。

- **视觉**
  - 图片上可叠一层从下到上的渐变遮罩（`linear-gradient(transparent 30%, rgba(0,0,0,0.5))`），保证白字/浅字可读。
  - 可选：悬停时图片轻微放大（`scale(1.02)`）+ 过渡 0.3s。

- **交互**
  - 整块可点击进入详情；可提供「跳过 / 下一则」小按钮供快速切换头条。

### 4.2 信息卡片（图文）

**结构**：图（上或左）+ 标题 + 摘要 + 元信息（来源、时间、标签）。

- **尺寸**
  - 大卡：图区域高度 200–240px，标题 H2。
  - 中卡：图 120–160px，标题 H3。
  - 小卡/列表项：图 80–96px（或仅占左侧），标题 H3/Body。

- **样式**
  - 背景：`--color-bg-card`，圆角 12px，边框用 `--color-border` 或无色。
  - 阴影：默认 `0 1px 3px rgba(0,0,0,0.06)`；悬停见下。

- **悬停（Hover）**
  - 卡片：`box-shadow: 0 8px 24px rgba(0,0,0,0.08)`；`transform: translateY(-2px)`；`transition: 0.25s ease`。
  - 图片：轻微放大 `scale(1.03)`，`overflow: hidden` 裁切。
  - 标题：主色高亮（如 `--color-primary`）或保持字色、仅加强字重。
  - 深色模式下阴影改为 `0 8px 24px rgba(0,0,0,0.3)`。

- **状态**
  - 未读：左侧或标题前小圆点（主色）；已读：无点缀或弱化字色。
  - 收藏/星标：图标用辅助色或霓虹辅色。

### 4.3 导航栏（Glassmorphism）

**位置**：顶部横条 或 左侧边栏（可折叠）；移动端顶部 + 底 Tab 或抽屉。

- **毛玻璃效果**
  - `background: rgba(255,255,255,0.72)`（浅色）或 `rgba(20,20,24,0.85)`（深色）。
  - `backdrop-filter: blur(12px)`；`-webkit-backdrop-filter: blur(12px)`。
  - 底部细线：`border-bottom: 1px solid rgba(0,0,0,0.06)`（浅）或 `rgba(255,255,255,0.08)`（深）。

- **尺寸与内容**
  - 高度：56px（顶栏）或全高（侧栏）；Logo + 主导航 + 搜索/用户区。
  - 激活项：主色字或左侧竖条（4px 主色）+ 字重 600。

- **滚动行为**
  - 可选：滚动超过 80px 后顶栏由透明变为毛玻璃，或增加轻微阴影，增强「浮起」感。

## 五、Bento Grid 布局建议

- **桌面端**
  - 首行：1 个大块（Hero，占 2/3 宽）+ 1 个小块（次头条，1/3 宽）。
  - 第二行：3 个等宽焦点卡 或 2+1（左二右一）。
  - 第三行起：2–3 列等宽流式卡片，或 1 列列表 + 右侧推荐/筛选。

- **栅格**
  - 12 列栅格，间距（gutter）24px；卡片与卡片间距 ≥ 24px。
  - 移动端：单列或 2 列小卡，gutter 16px。

- **实现**
  - CSS Grid：`grid-template-columns: repeat(12, 1fr)`，子项用 `grid-column: span n` 控制占位。
  - 或使用 Ant Design Grid / 自定义 Bento 组件，按「占几列」配置。

## 六、微交互动画（提升高级感）

### 6.1 卡片悬停 — 轻微上浮 + 阴影

- **触发**：鼠标移入信息卡片。
- **效果**：`transform: translateY(-4px)`；阴影从 `0 2px 8px` 变为 `0 12px 32px rgba(0,0,0,0.1)`；过渡 0.25s ease。
- **价值**：明确可点击、层次感强，且不花哨。

### 6.2 列表/流式进入 — 淡入 + 上移

- **触发**：内容进入视口（如滚动加载、Tab 切换）。
- **效果**：`opacity: 0` → `1`，`transform: translateY(12px)` → `0`；`transition: 0.4s ease`；可为每个卡片加 `animation-delay: i * 0.05s` 形成短序列感。
- **价值**：信息流「顺滑出现」，减少生硬弹出。

### 6.3 全局加载 / 骨架 — 呼吸式骨架

- **触发**：列表或详情加载中。
- **效果**：骨架块使用 `background: linear-gradient(90deg, #eee 25%, #f5f5f5 50%, #eee 75%)` 且 `background-size: 200% 100%`，`animation: shimmer 1.2s ease-in-out infinite`（位移背景），深色模式改为深灰渐变。
- **价值**：等待时不空白、不闪屏，体现细节。

## 七、实施检查清单

- [ ] 引入字体（如 Noto Sans SC）并在 `body` 上应用字号层级。
- [ ] 将配色写入 CSS 变量（见 `design-tokens.css`），并支持浅/深主题切换。
- [ ] 导航栏应用毛玻璃（`backdrop-filter` + 半透明背景）。
- [ ] Hero 与信息卡片使用统一圆角、阴影与悬停规范。
- [ ] Bento 首屏用 Grid 实现 1 大 + 1 小 或 2+1 布局。
- [ ] 列表/卡片进入视口时加淡入上移动画（可选用 Intersection Observer）。
- [ ] 加载态统一为骨架屏 + 呼吸动画。

## 八、设计令牌与主题切换

- **CSS 变量**：见前端项目 `src/design-system/design-tokens.css`，包含浅色（莫兰迪）、深色（霓虹）两套变量及字号、间距、圆角、过渡时间。
- **默认主题**：未设置时使用浅色（`:root` 与 `.theme-light` 同值）。
- **切换深色**：在 `<html>` 或 `<body>` 上添加 class `theme-dark` 即可；移除则恢复浅色。
- **使用方式**：组件内直接使用 `var(--color-primary)`、`var(--space-lg)`、`var(--text-h2-size)` 等；导航栏使用 `.ds-nav-glass`，卡片使用 `.ds-card`，骨架使用 `.ds-skeleton`，列表进入动画使用 `.ds-fade-up`。
